Veröffentlicht am 2017-07-31
hexo+github搭建博客

  1. 申请github账号，新建仓库，仓库用户名为用户名.github.io。

  2. 安装nodejs和git

    • 安装nodejs
      sudo apt-get install nodejs
      sudo apt-get install npm
    • 安装后的nodejs版本太低，要更新，先更新npm，再更新nodejs
      sudo npm install -g npm
      #此时会出现apt-get安装后没有添加链接#
      ln -s /usr/bin/nodejs /usr/bin/node
      #更新nodejs#
      npm install -g n
      sudo n stable
    • 安装git
      sudo apt-get install git
    • 安装hexo
      sudo npm install --unsafe-perm --verbose -g hexo

  3. hexo使用

    • 创建博客文件夹
      hexo init blog
      cd blog
      nom install #安装依赖

    • git配置
      配置SHH key

      cd ~/.ssh

      检查本机存在的SSH密钥，如果没找到，说明是第一次使用

      ssh-keygen -t rsa -C "注册github的email"

      连续敲三下回车，会在用户目录下生成一个文件，打开用户目录，找到.shh\id_rsa.pub文件，复制里面内容，到github个人设置里面的SSH and CPG keys，然后new一个SSH key。里面的title随便写，将复制的内容铁道key里面。测试是否成功可以用ssh -T github@github.com。成功之后还需要再配置

      git config --global user.name "github用户名"
      git config --global user.email "github的注册邮箱"

      这样子就可以免密登录github

    • 修改配置文件
      讲blog下的_config.yml里面的Deployment部分进行修改
      deploy:
      type: git
      repo: git@github.com:username/username.github.io.git
      branch: master
      type、repo、branch前面有两个空格，：后面有一个空格。

    • 安装git插件
      npm install hexo-deployer-git --save

    • hexo常用命令
      新建博文
      hexo new post "博客文件名"
      编辑/source/_post/博客文件名，使用markdown语法。

      编辑完成后
      生成静态页面
      hexo g：hexo generate #编译成静态文件
      部署hexo
      hexo d：hexo deploy #部署网站
      可在本地服务器预览
      hexo s：hexo server #本地运行
      清除生成内容
      hexo c：hexo clean #清空generate生成器的文件
